Commenting

To avoid confusion, criticisms have been disabeled for now. Look for this feature to be re-introduced in the future

Most links and text posts allow user to submit comments about the link or post. Unlike Reddit, Sciteit allows for two different types of reply; comments and criticisms. A comment is a general point of discussion or question about the link. A criticism is meant to be used far more rarely and only when you feel there is an outstanding issue with the content of what was posted. If you imagine you were a referee on a paper, a criticism would be something that would prevent you from recommending the article is published. A comment is everything else. For more information on criticisms, see the criticism help page.

Submitting and viewing

To submit a comment, simply click on the comment button below any link. You can then either submit a new comment using the box at the top of the page, or reply to an existing comment. Comments made in reply to other comments are moved around with the comment they are replying to, making a "comment tree". In addition to plain text, Sciteit supports a number of extra formatting tools, which are described in the formatting help. Sciteit currently supports basic markdown syntax, with support for embedding simple latex.

Viewing and sorting

Users vote on comments the same as they do links, hopefully ensuring that the most relevent comments are shown first. The comment tree is sorted by the algorithm selected at the top of the page. There are several different sorting algorithms that have different behaviours. These are described in detail in this excellent blog post. If in doubt, the default sort of "best" is probably the best (who would have thought it?).
